## 十五、Web UI 结构

### PC 端布局

```
┌──────────────────────┬─────────────────────────────────┐
│ 侧边栏               │ 主区域                           │
│                      │                                  │
│ ▼ 项目（可折叠）      │  会话激活时：聊天视图             │
│   日常事务 ●         │  任务激活时：任务详情             │
│   MelodySync 开发    │  （同一区域切换）                 │
│   读书计划           │                                  │
│ ──────────────────── │                                  │
│ [会话]  [任务]        │                                  │
│                      │                                  │
│  会话列表 / 任务列表  │                                  │
└──────────────────────┴─────────────────────────────────┘
```

### 移动端布局

```
┌─────────────────────────────────────┐

├─────────────────────────────────────┤

│                                     │

```

### 交互规则

1. **默认状态**：进入应用默认选中"日常事务"项目，显示会话 tab
2. **项目切换**：切换项目时保持当前 tab 状态（看会话就还是会话，看任务就还是任务），用户思维不断
3. **主区域**：点击会话进入聊天视图，点击任务进入任务详情，同一区域切换
4. **PC 项目列表**：默认展开，可折叠以获得更多内容空间
5. **移动端项目**：横向滚动 tab，当前项目高亮

### 任务列表（侧边栏任务 tab）

参考微信待办风格，每条任务显示：

```
○  每日晨报                    09:00 ↺
○  需要确认预算                 2月19日   ← 红色（人类待处理）
☑  竞品调研                    完成
```

- 左侧圆形 checkbox（仅 assignee=human 任务可勾选完成）
- 右侧时间：recurring 显示下次执行时间，scheduled 显示执行日期，human 任务显示创建日期（红色）
- recurring 任务右下角显示 ↺ 图标
- 按 bucket 分组：**人类待处理**（高亮）/ **AI 周期任务** / **AI 一次性任务**
- 每条任务右侧有 enable/disable toggle（关闭后任务变灰，显示"已暂停"标签）
- 关闭的任务仍然显示在列表中（不隐藏），只是变灰

### 任务编辑面板（点击 [编辑] 后弹出）

参考 v1 的 persistent-editor-ui，字段按 assignee + kind 组合显示：

**所有任务共有字段：**
- 标题
- 描述
- 归属项目
- 启用/禁用 toggle（enabled）

**AI 任务字段：**
- 执行器类型（ai_prompt / script / http）
- 执行动作（prompt 文本，ai_prompt 时显示）
- Shell 命令（script 时显示）
- HTTP 配置（http 时显示）
- 执行服务（tool / model 选择）
- 知识库路径（memoryPath，可覆盖项目默认值）
- 是否注入上次执行结果（includeLastLog toggle）

**调度字段（kind=scheduled/recurring 时显示）：**
- recurring：循环节奏（每天/每周/每小时）+ 执行时间 + 星期几
- scheduled：定时时间（datetime-local）

**人类任务字段：**
- 完成说明（waiting_instructions，只读，AI 填写）
- 来源任务（只读）
